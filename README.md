# Digital Portfolio (Next.js + Payload)

Modern, data-driven personal portfolio built with Next.js 15, React 19, Tailwind CSS v4, and Payload CMS. The site highlights professional experience, showcases collaborative government and personal projects, and embeds DevOps stories including home-lab deployments and IaC workflows.

> **Live Demo:** https://portfoliocf.harrisviewcodes.uk/  
> **Legacy v1:** https://legacy1portfoliocf.harrisviewcodes.uk/  
> **Admin (local dev):** http://localhost:3000/admin

---

## Highlights

- **Component-first UI** – Reusable React components (e.g., `ProjectDetail`, `HorizontalCard`, `PreviousExperience`) keep layout and data concerns separate.
- **Rich motion design** – Framer Motion variants animate cards, navigation, and section reveals for a polished feel.
- **Two and more theme presets** – Light and dark with a custom `ThemeProvider` and CSS tokens in `src/app/(frontend)/theme/*`.
- **Payload ready** – Users and Media collections plus autogenerated admin routes (`/app/(payload)`) allow the portfolio to evolve into a CMS-backed experience.
- **End-to-end testing hooks** – Vitest for integration tests and Playwright for E2E coverage are preconfigured.

## Getting Started

### Prerequisites

- Node.js 18.20+ or 20.9+
- pnpm 9 or 10 (`corepack enable`)
- MongoDB instance (local or cloud) for Payload collections

### Setup

1. **Install dependencies**
   ```bash
   pnpm install
   ```
2. **Configure environment** – copy `.env` (or set your own) with:
   ```env
   DATABASE_URL=mongodb://127.0.0.1/digital-portfolio
   PAYLOAD_SECRET=replace-with-long-random-string
   ```
3. **Run the dev server**
   ```bash
   pnpm dev
   ```
4. Visit `http://localhost:3000/home` for the portfolio and `http://localhost:3000/admin` to bootstrap the first Payload admin user.

### Payload Utilities

- `pnpm generate:types` – refresh `src/payload-types.ts` after schema edits.
- `pnpm generate:importmap` – regenerate the admin import map when adding custom components.

## Scripts

| Command        | Description                                               |
| -------------- | --------------------------------------------------------- |
| `pnpm dev`     | Start Next.js + Payload in development mode               |
| `pnpm devsafe` | Clean `.next` then run the dev server                     |
| `pnpm build`   | Production build (`next build`)                           |
| `pnpm start`   | Start the compiled app (`next start`)                     |
| `pnpm lint`    | Run ESLint across the project                             |
| `pnpm test`    | Executes `test:int` (Vitest) then `test:e2e` (Playwright) |
| `pnpm payload` | Run Payload CLI commands (migrations, seeds, etc.)        |

> Tip: run `tsc --noEmit` after large refactors to ensure the generated payload types stay in sync.

## Content & Customization

- `src/data/HomeInfo.tsx` – hero content, education, certificates, testimonials, contact links.
- `src/data/ExpInfo.tsx` – work history, project badges, and detail page routes.
- `src/data/ProjectsData.ts` – cards for collaborative vs personal projects plus tool imagery.
- `src/data/ProjectInfo.tsx` – long-form project case studies powering `/projects/*` pages.
- `src/data/AvatarInfo.tsx` – avatar text, rotating titles, and social/contact icons.
- `src/components/ProjectDetail.tsx` – single entry point for rich project narratives (links, tech stacks, sections).

Update these files to refresh copy or add new projects without touching layout code.

---

## Theming & Styling

- Theme-specific overrides live in `src/app/(frontend)/theme/{light,dark,sepia,monochrome}.css` and are toggled through `ThemeProvider` + `ThemeToggler`.
- Global tokens (color palette, typography) are defined in `src/app/(frontend)/globals.css` using Tailwind CSS v4 `@theme` blocks.
- Components lean on CSS variables and utility classes, keeping animations (Framer Motion) and layout responsive across breakpoints.

## Testing & Quality

- `pnpm test:int` – Vitest suite for component or hook logic (runs in Node + jsdom).
- `pnpm test:e2e` – Playwright tests (configure credentials/URLs as needed).
- `pnpm lint` – ESLint with `eslint-config-next` for consistent conventions.
- `tsc --noEmit` – optional sanity check for typed code after large edits.

---

## Deployment Notes

- Set `DATABASE_URL` and `PAYLOAD_SECRET` in the target environment (Cloudflare Pages + external DB, Vercel, etc.).
- The provided Dockerfile expects `next.config.mjs` to enable `output: 'standalone'`. Add that before building container images.
- Payload’s admin routes are bundled inside Next.js; no separate server is required.
